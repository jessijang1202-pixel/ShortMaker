# 간단영상 자동생성 체크리스트 (Auto-Gate Mode)

> 이 파일은 **사용자 개입 없이** 주제 입력만으로 완성까지 자동 생성되는
> "간단영상" 파이프라인에서 사용합니다.
> AI는 각 단계 산출물을 만든 직후, 다음 단계로 넘어가기 전에
> 반드시 스스로 이 기준으로 판정하고 결과를 표로 출력해야 합니다.
> **잘 맞추면 통과가 아니라 재시도입니다.** 사람이 걸러줄 안전장치이므로 기준을 관대하게 적용하지 마세요.

---

## 게이트 1: 스크립트 생성 직후
스크립트를 만든 즉시, 아래 항목을 전부 판정한 뒤 다음 단계(스토리보드)로 넘어갈지 결정하세요.

| 항목 | 기준 | 판정 |
|---|---|---|
| 훅 배치 | 첫 3초(첫 1~2문장) 안에 시청자 관심을 잡는 훅이 있는가 | pass/fail |
| 러닝타임 적합성 | 목표 길이(30초=75~90단어 등) ±10% 이내인가 | pass/fail |
| 문장당 정보 밀도 | 한 문장에 정보 1개만 담겨 있는가 (과밀 없음) | pass/fail |
| CTA 존재 | 마지막에 행동 유도 문구가 있는가 | pass/fail |
| 금칙어/저작권 | 브랜드명·가사·유명인 실명 등이 없는가 | pass/fail |

**판정 규칙**: 5개 중 1개라도 fail → 해당 항목만 지시해서 스크립트 재생성 (전체 재생성 금지, 문제 문장만 수정). 재생성 2회 초과 시 "보류" 상태로 전환하고 로그에 사유 기록.

---

## 게이트 2: 스토리보드 생성 직후

| 항목 | 기준 | 판정 |
|---|---|---|
| 씬-문장 매핑 | 씬 시간 배분이 스크립트 문장 흐름과 논리적으로 대응하는가 | pass/fail |
| 소스 확보 가능성 | 각 씬의 이미지/영상 소스가 저작권 문제 없이 확보 가능한가 | pass/fail |
| 자막 길이 | 씬별 자막 오버레이가 12자 내외를 넘지 않는가 | pass/fail |
| 세로 안전 영역 | 9:16 기준 핵심 요소가 safe zone 안에 배치되는가 | pass/fail |

**판정 규칙**: fail 항목이 있으면 해당 씬만 재생성. 이미지 소스 확보 불가 판정이 나오면 대체 이미지 카테고리를 AI가 스스로 제안하고 재시도.

---

## 게이트 3: 스타일/키 적용 직후

| 항목 | 기준 | 판정 |
|---|---|---|
| 브랜드 프리셋 일치 | 폰트/색상/자막 스타일이 지정된 프리셋과 일치하는가 | pass/fail |
| BGM 키 일치 | 음악/효과음 분위기가 콘텐츠 키와 맞는가 | pass/fail |
| 키 급변 여부 | 최근 생성된 영상들과 비교해 키가 튀지 않는가 | pass/fail |

---

## 게이트 4: 렌더링 직전 최종 검증

| 항목 | 기준 | 판정 |
|---|---|---|
| 러닝타임 | 영상 길이가 목표 범위 ±10% | pass/fail |
| 자막 싱크 | 자막 타이밍이 음성/영상과 맞는가 | pass/fail |
| 플랫폼 규격 | 해상도/비트레이트가 대상 플랫폼 기준 충족 | pass/fail |

**최종 판정**: 4개 게이트 모두 pass → 렌더링 진행. 하나라도 fail → 자동 재생성 루프. 동일 게이트에서 2회 연속 fail → 사용자에게 "자동 생성 실패, 수동 확인 필요" 알림 후 보류.

---

### 구현 메모
- 각 게이트는 독립 함수/프롬프트 호출로 분리해서, 실패 시 해당 단계만 재실행하는 것으로 구성하세요 (전체 파이프라인 재시작 금지).
- 재생성 최대 횟수(예: 게이트당 2회)를 반드시 두세요. 무한 루프 방지 + 비용 관리.
- 판정 결과는 로그로 남겨서, 어떤 유형의 fail이 잦은지 추적하면 프롬프트 개선 포인트를 찾기 쉬워집니다.
