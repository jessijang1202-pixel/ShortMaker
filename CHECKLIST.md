# 영상 제작 체크리스트

SnapReel이 생성하는 숏폼 영상의 품질 기준을 정리한 문서입니다.
새 기능을 추가하거나 프롬프트를 수정할 때 이 목록을 함께 점검하세요.

이 문서는 3부로 구성됩니다.
1. **실제 코드에 구현된 규칙** — 지금 앱이 실제로 지키고 있는 기준
2. **간단 영상 참고 체크리스트** — 자동 생성 파이프라인 설계 시 참고용 (게이트 방식, 현재 UI로는 미구현)
3. **고급 영상 참고 체크리스트** — 10단계 사용자 확인 흐름 설계 시 참고용 (팝업 UI는 2026-07 제거됨)

---

## 1부. 실제 구현된 규칙

### 1-1. 인트로(0~8초, Veo AI 클립) 자막

> 2026-07 추가 — 훅 구간은 영상의 첫인상이라 자막 임팩트가 가장 중요합니다.

- [ ] 자막을 8초 내내 한 줄로 길게 띄우지 않는다 (읽다가 지루해짐)
- [ ] 3~6개의 짧은 캡션 조각(2~5단어)으로 나눠 순차적으로 노출한다
- [ ] 각 조각의 시간 구간(start~end)이 0초부터 duration까지 빈틈없이 이어진다
- [ ] 가장 강력한 키워드가 담긴 조각을 맨 앞(첫 1초 안)에 배치한다
- [ ] 자막 스타일이 일반 본문 폰트가 아니라 굵고(font-black) 색이 강한(브랜드 컬러 배경 칩) 형태로 강조된다
- [ ] 데이터: `VeoCoreClip.captions: CaptionBurst[]` — AI가 안 주면
      `splitIntoCaptionBursts()`로 자동 폴백 분할된다 (누락 없이 항상 존재)

구현 위치:
- 프롬프트 규칙: `src/prompts/index.ts` → `buildScriptSplitPrompt()`
- 파싱/폴백: `src/services/gemini.service.ts` → `generateScriptSplit()`,
  `src/utils/captionBursts.ts`
- 렌더링: `src/components/steps/StoryboardStep.tsx` (미리보기),
  `src/components/steps/VeoClipStep.tsx` (고급 모드 편집 화면)

### 1-2. 후반부(8~30초, 슬라이드 6개) 자막

- [ ] 화면 텍스트(`on_screen_text`)는 1~2줄, 한눈에 읽히는 길이
- [ ] 자막 위치/크기/스타일은 레퍼런스 분석 결과(`subtitleNarration`)를 따른다
      — 없으면 트렌드 기본값(`TREND_DEFAULT_STYLE`) 사용
- [ ] `subtitleEnabled`가 꺼져 있으면 자막을 아예 표시하지 않는다

### 1-3. 레퍼런스 스타일 반영

- [ ] 레퍼런스 영상을 올렸다면 컨셉·색감·자막 위치/크기/스타일·BGM·효과음이
      실제 결과물(대본 톤, 슬라이드 이미지, 자막)에 반영된다
- [ ] 레퍼런스가 없으면 최신 숏폼 트렌드 스타일로 자동 대체된다

### 1-4. 영상 길이/구성

- [ ] 전체 길이: 30초 (Veo 8초 + 슬라이드 22초, 6개 장면)
- [ ] 각 슬라이드 씬: 3~4초 내외

### 1-5. 톤/안전성

- [ ] 기획 단계의 "피해야 할 표현"(bannedExpressions)이 대본에 등장하지 않는다
- [ ] 확인되지 않은 사실을 단정하지 않는다
- [ ] 브랜드 로고·특정 심벌·타 브랜드명이 프롬프트/화면에 노출되지 않는다

## 2부. 간단 영상 — 자동 게이트 체크리스트 (참고용)

> 사용자 개입 없이 주제 입력만으로 완성까지 자동 생성되는 파이프라인을 설계할 때 참고.
> AI가 각 단계 산출물을 만든 직후 스스로 판정하고, 기준 미달이면 자동 재생성하는 방식.
> **잘 맞추면 통과가 아니라 재시도입니다.** 안전장치이므로 기준을 관대하게 적용하지 않습니다.

### 게이트 1: 스크립트 생성 직후

| 항목 | 기준 |
|---|---|
| 훅 배치 | 첫 3초(첫 1~2문장) 안에 시청자 관심을 잡는 훅이 있는가 |
| 러닝타임 적합성 | 목표 길이 ±10% 이내인가 |
| 문장당 정보 밀도 | 한 문장에 정보 1개만 담겨 있는가 (과밀 없음) |
| CTA 존재 | 마지막에 행동 유도 문구가 있는가 |
| 금칙어/저작권 | 브랜드명·가사·유명인 실명 등이 없는가 |

fail 시: 해당 항목만 지시해서 재생성 (전체 재생성 금지). 2회 초과 시 "보류" 전환 + 로그 기록.

### 게이트 2: 스토리보드 생성 직후

| 항목 | 기준 |
|---|---|
| 씬-문장 매핑 | 씬 시간 배분이 스크립트 흐름과 논리적으로 대응하는가 |
| 소스 확보 가능성 | 이미지/영상 소스가 저작권 문제 없이 확보 가능한가 |
| 자막 길이 | 씬별 자막 오버레이가 12자 내외를 넘지 않는가 |
| 세로 안전 영역 | 9:16 기준 핵심 요소가 safe zone 안에 배치되는가 |

fail 시: 해당 씬만 재생성. 소스 확보 불가 시 대체 카테고리를 AI가 스스로 제안 후 재시도.

### 게이트 3: 스타일/키 적용 직후

| 항목 | 기준 |
|---|---|
| 브랜드 프리셋 일치 | 폰트/색상/자막 스타일이 지정된 프리셋과 일치하는가 |
| BGM 키 일치 | 음악/효과음 분위기가 콘텐츠 키와 맞는가 |
| 키 급변 여부 | 최근 생성된 영상들과 비교해 키가 튀지 않는가 |

### 게이트 4: 렌더링 직전 최종 검증

| 항목 | 기준 |
|---|---|
| 러닝타임 | 영상 길이가 목표 범위 ±10% |
| 자막 싱크 | 자막 타이밍이 음성/영상과 맞는가 |
| 플랫폼 규격 | 해상도/비트레이트가 대상 플랫폼 기준 충족 |

모두 pass → 렌더링 진행. 동일 게이트 2회 연속 fail → 사용자에게 "자동 생성 실패, 수동 확인 필요" 알림 후 보류.

구현 시 유의점: 게이트별로 독립 함수/프롬프트로 분리해 실패 시 해당 단계만 재실행. 게이트당 최대 재생성 횟수를 두고(예: 2회) 무한 루프를 방지. 판정 결과는 로그로 남겨 fail 유형을 추적하면 프롬프트 개선 포인트를 찾기 쉬움.

## 3부. 고급 영상 — 10단계 사용자 확인 체크리스트 (참고용)

> 매 단계마다 사용자가 승인하는 흐름을 설계할 때 참고.
> 이 체크리스트를 기반으로 `StepReviewPanel.tsx` 팝업을 만들었으나,
> 2026-07 사용자 요청("팝업창이 불필요하다, 스스로 체크해서 수정해줘")에 따라 팝업 UI는 제거됨.

각 단계 확인 형식: **확인 요청** (무엇을 확인해야 하는지) → **이유** (체크리스트 근거) → **선택지** (승인 / 수정 요청 / 다시 만들기)

| 단계 | 체크 항목 | 확인 요청 예시 |
|---|---|---|
| 1. 주제/컨텍스트 | 주제가 한 문장으로 명확한가 · 핵심 메시지가 15초 이내로 요약되는가 | "핵심 메시지를 'ㅇㅇ'로 잡았는데 이 방향이 맞을까요?" |
| 2. 타겟/키 설정 | 타겟 청중(연령대/관심사)이 지정되었는가 · 키(정보성/유머/감성) 프리셋이 선택되었는가 | "타겟은 20대 직장인으로 잡고 유머 키로 갈까요?" |
| 3. 스크립트 이슈 | 훅이 첫 3초 안에 배치되어 있는가 · 러닝타임이 목표 범위 안인가 | "오프닝 한 문장이 두 가지입니다. 어느 쪽이 더 끌리나요?" |
| 4. 스크립트 확정 | 문장당 정보 밀도가 적절한가 · CTA 포함 여부 · 저작권·초상권 위험 단어 없음 | "이 문장에 브랜드명이 언급되는데 그대로 둘까요?" |
| 5. 씬 분할/스토리보드 | 씬 시간이 스크립트 흐름과 논리적으로 맞는가 | "6문장을 5개 씬으로 나눴습니다. 자연스러운가요?" |
| 6. 비주얼 소스 선정 | 각 씬 이미지/영상 소스가 저작권 문제 없는가 | "3번 씬 이미지 소스 후보 3개 중 골라주세요." |
| 7. 텍스트 오버레이/자막 | 씬별 자막 길이가 12자 이내인가 · safe zone 안에 배치되는가 | "이 씬 자막이 길어서 잘릴 수 있어요. 줄일까요?" |
| 8. 스타일(폰트/컬러) 적용 | 브랜드 프리셋과 일치하는가 | "폰트를 프리셋 기본값으로 적용했어요. 바꿀까요?" |
| 9. BGM/효과음 적용 | 음악 분위기가 콘텐츠 키와 맞는가 · 이전 영상과 키가 급격히 다르지 않은가 | "선택한 BGM이 지난 영상보다 빠른데, 진행할까요?" |
| 10. 최종 기술 검증 | 러닝타임 ±10% · 자막 타이밍 일치 · 해상도/비트레이트 플랫폼 기준 충족 | "최종 길이가 32초로 목표보다 2초 깁니다. 렌더링할까요?" |

구현 시 유의점: 이전 단계 사용자 응답을 반영한 뒤에만 다음 단계 프롬프트 생성. "다시 만들기" 선택 시 해당 단계만 재생성하고 이전 단계 결과물은 유지(전체 재생성 금지). 사용자가 여러 단계를 건너뛰고 싶어 하면 남은 단계는 2부의 자동 게이트 로직으로 전환하는 것도 고려.
